# 最小人工验收脚本

## 目标

验证本轮维护没有破坏主链路，并覆盖已修复的关键回归点。

## 验收前准备

1. 在设置页填写主模型与问题模型的真实 `API Key`、接口地址和模型名。
2. 确认欢迎页“使用提示”在模型已配置后不再自动弹出。
3. 打开浏览器控制台，确认没有明显运行时报错。

## P0 主链路

1. 新建对话，发送一条普通问题。
   预期：用户消息立即入列，assistant 开始流式输出，最终正常收尾。

2. 在 assistant 正常回复后，刷新页面并重新进入刚才的历史对话。
   预期：历史消息完整恢复，消息顺序不乱，没有重复首屏欢迎语。

3. 将主模型配置改成错误的 `API Key` 或错误接口，再发送一条消息。
   预期：聊天区出现原位失败态 assistant 消息，不再只弹 `alert`。

4. 在失败态 assistant 消息上点击“重试”，并修正模型配置。
   预期：原消息位置重新请求，失败态被新回复替换，不会错位重试其他消息。

5. 找到一条历史里的用户消息，点击“重试”。
   预期：从该用户消息开始重新生成后续 assistant 内容，命中正确上下文。

6. 触发至少一个问题卡，并完成一次答案选择。
   预期：问题卡可正常生成，选择答案后会插入答题卡 UI，不会破坏主消息重试。

7. 完成问题卡全部作答，等待自动总结回到主对话。
   预期：总结用户消息与新的 assistant 回复都会写入历史，刷新后仍可恢复。

8. 打开并关闭答案卡浮层。
   预期：不再出现 `deselect is not defined` 一类运行时错误。

## P1 补充验证

1. 模型未配置时进入首页。
   预期：欢迎页显示设置提示。

2. 点击“稍后再说”关闭提示后刷新页面。
   预期：在仍未配置的情况下，本地已关闭标记生效，不会每次都重复弹出。

3. 通过快速提示创建新对话。
   预期：会直接发送所选 prompt，不会因为闭包/时序问题发送空消息或旧消息。
