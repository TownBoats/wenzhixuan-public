# 升级方案 —— 2026 年 5 月 1 日

## 背景

当前请求/响应链路基于手写的 `fetch` + 手动 SSE 解析，缺乏对思考模式（reasoning tokens）的支持。本次升级目标：

1. **迁移至 Vercel AI SDK**：以 `streamText` + `@ai-sdk/openai` 替换 `AgentModel` 中的原始 HTTP 层
2. **支持思考模式内容**：为 DeepSeek R1、Qwen QwQ、Claude 扩展思考等具备推理能力的模型，在完整数据链路中增加 `thinking` 通道

两项工作在架构上解耦，可独立或同时实施。

---

## 一、Vercel AI SDK 迁移

### 技术前提与约束

本项目是**纯前端 BYOK 应用**，无后端。`@ai-sdk/openai` 的 `createOpenAI` provider + `streamText` 可以在浏览器环境直接运行（本质仍是 fetch，只是封装了 SSE 解析和类型化流）。

### URL 映射问题（最大技术风险）

`createOpenAI({ baseURL })` 会在 `baseURL` 末尾自动拼接 `/chat/completions`，而 `ConfigManager.buildEndpointUrl()` 返回的是包含路径的完整 URL。

**解决策略**：在 `AgentModel` 中做一次 URL 变换：

```
完整URL:       https://api.deepseek.com/v1/chat/completions
              ↓ 去掉结尾的 /chat/completions
aiSdkBaseURL: https://api.deepseek.com/v1
```

对不以 `/chat/completions` 结尾的非标准端点，退化为通过 `fetch` override 直接传入完整 URL，保证兼容性：

```javascript
_buildAiSdkBaseUrl(fullUrl) {
  const absolute = fullUrl.startsWith('http')
    ? fullUrl
    : `${window.location.origin}${fullUrl.startsWith('/') ? fullUrl : '/' + fullUrl}`;
  return absolute.replace(/\/chat\/completions(\?.*)?$/, '');
}
```

### 新的 `AgentModel.getCompletion()` 接口

**最小改动原则**：仅在第三个参数后扩展 `options`，不改变现有调用方的签名。

```javascript
// 现有调用方（useQuestionFlow，无感知变化）
optionAgent.getCompletion(contextMessages, (chunk) => { ... })

// runAssistantTurn 新调用（扩展 options 传入 onReasoning）
mainAgent.getCompletion(requestMessages, onChunk, onComplete, { onReasoning: feedThinking })
```

内部实现从手写 SSE 解析：
```
fetch → getReader → buffer.split('\n') → JSON.parse → delta.content → onDataReceived
```
换成 AI SDK 类型化流：
```
streamText(provider, messages) → fullStream async iterable
  → part.type === 'text-delta'  → onDataReceived(part.textDelta)
  → part.type === 'reasoning'   → options.onReasoning?.(part.textDelta)
  → part.type === 'finish'      → onComplete()
```

**保留的现有行为**：

| 行为 | 处理方式 |
|---|---|
| 3 次重试 + 指数退避 | 保留，AI SDK 不内置重试 |
| 打字机分块效果（10字符/10ms） | 保留，仅对 `text-delta` 启用；`reasoning` 不限速直接透传 |
| `onRequestStart` / `onResponseEnd` 回调 | 不变 |
| 60s 超时 + AbortController | 保留 |

### 新增依赖

```bash
pnpm add ai @ai-sdk/openai
```

## 二、思考模式支持

### 数据链路设计

思考内容（reasoning）**不经过** `StreamingParser`——它是与正文并行的独立流，不包含 `〖response〗` / `〖question〗` 标签。需要在主流之外开辟独立通道：

```
AI SDK fullStream
  ├── text-delta  → responseParser.feed()  → StreamingParser → currResponse[]
  └── reasoning   → feedThinking()         → currThinking string   ← 新增
```

### 各层级变更

#### `useStreaming.js`

新增 `currThinking` 状态与操作方法：

```javascript
const [currThinking, setCurrThinking] = useState('');
const currThinkingRef = useRef('');

const feedThinking = (chunk) => {
  setCurrThinking(prev => prev + chunk);
  currThinkingRef.current += chunk;
};

// reset() 中同步清空
currThinkingRef.current = '';
setCurrThinking('');
```

返回新增：`currThinking`、`currThinkingRef`、`feedThinking`。

#### `ChatPage.jsx` - `runAssistantTurn`

```javascript
// 1. 将 feedThinking 传入 getCompletion
await mainAgent.getCompletion(
  requestMessages,
  (content) => { streamedResponse += content; responseParser.feed(content); },
  () => {},
  { onReasoning: feedThinking }   // 新增
);

// 2. 流结束后快照 thinking 文本
const thinkingText = currThinkingRef.current;
const assistantUiContent = [
  ...(thinkingText ? [{ type: 'thinking', value: thinkingText }] : []),
  ...cloneUiContent(currResponseRef.current),
];
```

`thinking` item 排在 `response` / `question` 之前。

#### `uiMessages` 新 content type

| type | value 类型 | 写入场景 |
|---|---|---|
| `response` | `string` | 原有 |
| `question` | `string` | 原有 |
| `answer-card` | `{question, level, answer}` | 原有 |
| **`thinking`** | **`string`** | **新增，仅主 Agent 响应** |

#### `MessageBubble.jsx`

在 `renderContent()` 的 switch 中新增：

```javascript
case 'thinking':
  return (
    <ThinkingBlock
      key={index}
      content={item.value}
      isStreaming={isLoading && index === 0}
    />
  );
```

#### 新组件 `ThinkingBlock`

```
ThinkingBlock
├── 折叠态（默认）
│   ├── 流式进行中：[💭] 思考中... （脉冲动画）
│   └── 流式结束后：[💭] 查看思考过程  思考了 1,234 字  ▼
└── 展开态
    ├── header：[💭] 模型思考过程  ▲
    └── body：MarkdownRenderer（monospace 字体，浅灰/米黄背景，与正文区分）
```

#### 配置层：`ConfigManager.js`

`mainModelConfig` 扩展两个字段：

```javascript
{
  // ... 现有字段
  enableThinking: false,   // 是否启用思考模式
  thinkingBudget: 8000,    // 思考 token 预算（Anthropic 直连时有效）
}
```

`AgentModel` 在构建 `streamText` 时，若 `enableThinking: true` 且检测到 Anthropic 端点，追加：

```javascript
providerOptions: {
  anthropic: { thinking: { type: 'enabled', budgetTokens: this.settings.thinkingBudget } }
}
```

对 DeepSeek R1 / Qwen QwQ 等 OpenAI-compatible 思考模型：**无需额外配置**，AI SDK 自动从 `delta.reasoning_content` 提取并发出 `reasoning` part。

## 四、风险与对策

| 风险 | 说明 | 对策 |
|---|---|---|
| 非标准端点不兼容 | 用户配置了不以 `/chat/completions` 结尾的 URL | 检测后用 `fetch` override 直传完整 URL，绕过 SDK URL 拼接 |
| 打字机效果不均匀 | AI SDK delta chunk 大小由模型决定，可能比手写版更不均匀 | 保留现有 10char/10ms 分块逻辑，仅对 text-delta 生效 |
| thinking 内容为空 | 非思考模型不产生 `reasoning` part | `thinkingText` 为空时不写入 `uiContent`，UI 无感知 |
| 历史消息兼容 | 旧 chat history 无 `thinking` type item | `normalizeStoredChat` 透传未知 type；`MessageBubble` switch 有 `default` case 兜底 |
| Anthropic 直连思考 | Claude 扩展思考需要专用 header + 参数 | 初版通过 `providerOptions` 覆盖传递，暂不引入 `@ai-sdk/anthropic`（避免增加 bundle 体积） |
